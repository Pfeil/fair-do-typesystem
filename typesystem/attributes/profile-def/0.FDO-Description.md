# Uncertainty: 0.FDO/Description Localized Value Structure

## Issue

The specification states that `0.FDO/Description` provides "Human-readable and localized descriptions" but does not explicitly define:

1. **How localization is encoded** - Is it an object with `value` and `lang` fields? A language-tagged string?
2. **The exact JSON structure** when multiple languages are provided

## Current Implementation

Currently using a simple string syntax reference (`0.FDO/StringSyntax`). This allows validation but doesn't enforce proper localization structure.

## Possible Approaches

Same as documented in `0.FDO-Name.md`.

## Resolution Needed

Should be resolved consistently with `0.FDO/Name`.
