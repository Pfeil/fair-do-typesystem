# Uncertainty: 0.FDO/NumericInterval Format

## Issue

The specification states that `0.FDO/NumericInterval` provides "An interval for the value (space of real numbers)" but does not explicitly define:

1. **The exact format of the interval** - How are min/max bounds encoded?
2. **Whether bounds are inclusive or exclusive** - Is [0,1] different from [0,1)?
3. **How open-ended intervals are represented** - e.g., "greater than 5" or "at most 100"

## Current Implementation

Currently using a simple string syntax reference (`0.FDO/StringSyntax`). This allows validation but doesn't enforce proper interval structure.

## Possible Approaches

### Option A: String notation (mathematical)
```json
"0.FDO/NumericInterval": ["[0, 100]"]
```
Where `[` and `]` denote inclusive bounds, `(` and `)` denote exclusive bounds.

### Option B: Object with explicit fields
```json
"0.FDO/NumericInterval": [{
  "min": 0,
  "max": 100,
  "minInclusive": true,
  "maxInclusive": true
}]
```

### Option C: Separate min/max attributes
Use two separate values or a structured format like:
```json
"0.FDO/NumericInterval": ["0..100"]
```

## Resolution Needed

The specification should clarify:
- The canonical format for numeric intervals
- How to represent unbounded intervals (e.g., ">= 0")
- Whether the format differs for string length vs. numeric ranges
